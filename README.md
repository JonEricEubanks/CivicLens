## 🚀 Quick Start

```bash
# 1. Install dependencies
npm install

# 2. Set your GitHub token (for Azure AI inference)
cp .env.example .env
# Edit .env with your GITHUB_TOKEN

# 3. Start everything (API server + MCP server auto-launched)
npm start

# 4. Open the UI
open http://localhost:7072
```

> **Note:** `npm start` auto-forks the MCP data server on port 3000 — no second terminal needed.
> For Azure deployment: `azd up` (uses Bicep IaC in `infra/`).

## 💡 What It Does

CivicLens transforms raw municipal infrastructure data into **plain-language intelligence reports** that residents, council members, and journalists can understand.

**Try asking:**

| Query | What Happens |
|-------|-------------|
| *"What are the highest priority issues right now?"* | Weibull-ranked priority report with severity breakdown |
| *"Show me all school zone safety concerns"* | Proximity analysis for all schools with risk scores |
| *"Give me a status report for zone NW-3"* | Zone-specific dashboard with open/closed/pending counts |
| *"Dispatch Crew-A to work order WO-001"* | Action tool with confirmation modal + audit logging |
| *"Schedule an inspection at 123 Western Ave"* | Creates inspection record with email notification |

## 🏗️ Architecture

```
┌─────────────┐    ┌──────────────┐    ┌─────────────────┐    ┌──────────────┐
│   Intent    │───▶│    Data      │───▶│   Synthesis     │───▶│   Report     │
│   Agent     │    │    Agent     │    │   Agent + RAG   │    │   Agent      │
│  (LLM)      │    │  (MCP Tools) │    │  (LLM + KB)    │    │  (Formatter) │
└─────────────┘    └──────────────┘    └─────────────────┘    └──────────────┘
       🎯                📊                   ✍️                    📄
  Classifies        Fetches data        Generates report       Formats with
  intent &          from 15 MCP         grounded in RAG        data coverage,
  extracts          tools with          knowledge base         equity notes,
  filters           error recovery      (11 municipal docs)    audit trail
```

### Key Technical Features

| Feature | Description |
|:--------|:------------|
| **Multi-Agent Pipeline** | 5-stage pipeline: Intent → Data (ReAct) → RAG → Synthesis → Report with quality feedback loop |
| **MCP Protocol** | JSON-RPC 2.0 server with 16 tools (7 read + 3 action + 2 service request + 4 analytics) + RBAC |
| **RAG Knowledge Base** | 11 documents, 6 categories, TF-IDF + dense embedding support |
| **Weibull Decay Scoring** | Mathematical priority model: pothole k=1.8/λ=120, sidewalk k=2.2/λ=240 |
| **Action Tools** | Dispatch crews, update work orders, schedule inspections |
| **Predictive Analytics** | Weibull CDF forecasting, cost-of-inaction, what-if budget modeling |
| **Live Notifications** | SSE real-time status updates for service requests |
| **Chicago 311 Benchmarks** | Cross-city performance comparison using real 311 open data |
| **Outcome Tracking** | Resolution pipeline funnel, cost savings dashboard, before/after comparisons |
| **Data Persistence** | JSON files + Azure Cosmos DB (optional cloud sync) + audit log trail |
| **Security** | Rate limiting, input sanitization, RBAC, CSP, CORS allowlist, security headers |
| **Error Recovery** | Graceful degradation when MCP tools fail |

## 🤖 Responsible AI

CivicLens implements responsible AI practices throughout:

- **Evidence-Based Scoring** — Data coverage percentage replaces fabricated LLM "confidence" — measures how many data sources were actually consulted
- **Human-in-the-Loop** — Action confirmation modal before dispatching crews or scheduling inspections
- **Source Attribution** — Human-readable data source names with record counts
- **Audit Trail** — All actions logged to `mcp-server/data/audit-log.jsonl`
- **Data Equity** — Reports include equity disclosure about proximity-based prioritization
- **Limitations Disclosure** — Every report states analysis limitations
- **Input Guards** — 2000-char max, input validation throughout

<details>
<summary><b>Bias Testing & Fairness</b></summary>

CivicLens addresses geographic and socioeconomic bias risks inherent in municipal AI systems:

**Tested bias vectors:**
1. **Geographic equity**: Priority scoring weights (Weibull + severity + school proximity) are formula-driven, not LLM-generated. Verified that higher-income zones (NE-1, NW-3) do not receive systematically higher scores than lower-income zones.
2. **Resource allocation**: The `whatif_budget` tool models budget cut impacts per-zone, making disparate impact visible before decisions are made.
3. **Language bias**: Keyword fallback classifier supports non-complex English phrasing. Service request descriptions are stored verbatim, not LLM-summarized, preserving reporter voice.
4. **School proximity weighting**: Explicitly documented in REASONING.md. Schools in all zones receive equal proximity scoring — no zone is systematically advantaged.

**Mitigation strategies:**
- Data coverage metric is transparent: shows exactly which data sources informed each report
- Equity disclosure appears in every generated report
- Audit log captures all AI-driven actions for post-hoc fairness review
- Multi-model validation (gpt-4o-mini + Phi-3) reduces single-model classification bias
- Offline keyword classifier ensures service is accessible even without API, preventing technology-access bias

</details>

## 📐 Weibull Decay Priority Model

Priority scoring uses **Weibull survival analysis** to model how urgency increases non-linearly with age:

```
Score (0-200) = Severity(0-80) + WeibullAge(0-40) + SchoolProximity(0-30)
              + Traffic(0-20) + WeatherRisk(0-15) + TypeModifier(0-15)
```

| Infrastructure Type | Shape (k) | Scale (λ) | Behavior |
|:---|:---:|:---:|:---|
| **Potholes** | 1.8 | 120 days | Fast accelerating failure |
| **Sidewalks** | 2.2 | 240 days | Moderate decay |
| **Concrete** | 2.5 | 365 days | Slow bathtub curve |

> Parameters calibrated from APWA infrastructure decay curves for northern Illinois climate.

## 📚 Data Sources & Methodology

CivicLens is built on **real municipal reference data** — real schools, real regulatory standards, real cost benchmarks, and real infrastructure engineering models — packaged as a **pilot-ready deployment** that any municipality can adopt immediately.

<details>
<summary><b>What's Real</b></summary>

- **Schools are real**: Everett Elementary, Cherokee Elementary, Deer Path Middle School — actual Lake Forest School District 67 schools at their verified GPS coordinates
- **Standards are real**: APWA repair classifications, ASTM D6433 severity methodology, ADA compliance thresholds (>0.5″ heave = federal violation), RS Means 2024 cost data
- **The Weibull model is real**: Parameters calibrated from published infrastructure engineering survival analysis for northern Illinois climate conditions
- **Chicago 311 benchmarks are real**: Drawn from Chicago's Open Data Portal (data.cityofchicago.org) for cross-city performance comparison
- **The platform is live**: Deployed on Azure App Service at [civiclens-app.azurewebsites.net](https://civiclens-app.azurewebsites.net), fully operational for testing

</details>

<details>
<summary><b>Pilot Program Approach</b></summary>

The infrastructure issue records (potholes, sidewalk damage, work orders) use **representative sample data** calibrated against real municipal patterns — the standard approach for civic technology pilots before a formal municipal data-sharing agreement is in place. This is intentional: it lets developers and prospective municipal partners test every feature immediately without PII exposure or FOIA complications. The data layer abstraction (`lib/data-layer.js`) is designed for direct plug-in to any municipality's 311 system, GIS database, or work order platform — the agent pipeline, MCP protocol, and scoring engine are fully data-source agnostic.

</details>

<details>
<summary><b>Real-World Calibration</b></summary>

| Dataset | Real-World Basis |
|:--------|:-----------------|
| **Schools** | Real Lake Forest School District 67 names and locations — verified against district records |
| **Potholes** | Severity follows ASTM D6433 PCI methodology; size distributions match APWA 2023 Midwest data |
| **Sidewalks** | ADA compliance thresholds are federal regulatory standards; costs from 2024 RS Means |
| **Work Orders** | Resolution times benchmarked against Chicago 311 averages, adjusted for Lake Forest DPW staffing |
| **Chicago 311** | Real schema from Chicago Open Data Portal adapted for cross-city benchmarking |
| **Weibull Parameters** | k/λ values from published infrastructure engineering survival analysis literature |

</details>

<details>
<summary><b>Provenance Transparency</b></summary>

Every dataset includes machine-readable provenance metadata accessible via the `get_data_provenance` MCP tool (16th tool). This returns source models, calibration methods, and validation benchmarks for each dataset. See `mcp-server/data/data-provenance.json` for the full provenance manifest.

</details>
